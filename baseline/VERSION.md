## Format Derivation Chain

```
[Fusion 360] ──► [SolidWorks] ──► STEP (source CAD)
                                    │
                                    ├──► STL (tessellated mesh, from SolidWorks export)
                                    │
                                    └──► URDF (joint kinematics + STL references)
                                           │
                                           └──► USD (converted via urdf_to_usd.py using pxr API)
```

## Per-Format Details

### STEP (`step/`)
- **Source**: Manual export from SolidWorks
- **Available**: Right palm only (`right_palm_link.STEP`)
- **Note**: Individual finger STEP files not yet available from CAD team

### STL (`stl/`)
- **Source**: `wuji-hand-description` repo `meshes/{left,right}/` @ v0.2.2
- **Count**: 26 parts per hand (1 palm + 5 fingers x 5 links)
- **Format**: Binary STL, millimeter scale

### URDF (`urdf/`)
- **Source**: `wuji-hand-description` repo `urdf/{left,right}.urdf` @ v0.2.2
- **Mesh paths**: Modified internally by converter (original uses `../meshes/`)

### USD (`usd/`)
- **Generated by**: `scripts/urdf_to_usd.py`
- **Tool**: Python `pxr` (usd-core 26.3) on Python 3.12
- **Format**: USDA (ASCII, for readability and diffing)
- **Includes**: Mesh geometry (inline from STL), joint hierarchy, basic gray material
- **Does NOT include**: Textures, UV maps, PBR materials (baseline only)

## Regeneration

To regenerate USD from URDF + STL:
```bash
C:\Python312\python.exe scripts/urdf_to_usd.py --side both
```
